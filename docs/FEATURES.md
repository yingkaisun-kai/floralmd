# FloralMD 功能说明

这里汇总 FloralMD 目前提供的主要功能。

## 文件与窗口

- 从 Finder、菜单或命令行直接打开任意 `.md`、`.markdown` 和支持的纯文本文件；
- 不要求文件属于 Vault、工作区或特定文件夹；
- 使用 macOS 原生窗口标签页，在同一窗口切换不同目录中的文档；
- 支持自动保存、撤销、重做与未保存修改提醒；
- 支持从 Finder 双击 Markdown 文件打开 FloralMD。

## 编辑与阅读

- 行内实时渲染 Markdown，光标进入当前块时显示可编辑的原始标记；
- 提供编辑模式、源码模式和阅读模式；
- 源码模式使用等宽字体展示完整 Markdown；
- 阅读模式使用独立 HTML 渲染，并随文档修改和主题变化刷新；
- 支持查找、替换、拼写检查、文本缩放和常用 macOS 编辑快捷键；
- 支持 PDF 导出与系统打印。

## Markdown 语法

- 标题、粗体、斜体、粗斜体、删除线和高亮；
- 有序列表、无序列表、任务列表与嵌套列表；
- 引用、Callout、分隔线和脚注；
- 行内代码、围栏代码块与语法高亮；
- GFM 表格，包括列对齐、独立单元格换行和长字符串折行；
- 普通链接、自动链接、引用链接、图片和 Wiki Link；
- 行内数学公式与块级数学公式；
- HTML 标签与经过限制的原始 HTML 阅读渲染；
- 反斜杠转义、注释和常用 GFM 扩展。

## 文档导航

- 左侧大纲根据当前文档标题生成，并跟随阅读位置；
- 大纲可折叠为窄边缘入口；
- 右侧文件树显示当前 Git 工作区或当前目录中的 Markdown 文件；
- 文件夹显示递归 Markdown 数量，并自动展开到当前文件；
- 支持在文件树中切换文档；
- Git 状态颜色会从改动文件传播到父文件夹。

## Git 与长文档辅助

- 自动向上查找距离当前文件最近的 Git 仓库；
- 只读展示已修改、新增、删除和未跟踪文件；
- 编辑区左侧显示相对当前提交的改动标记；
- 列表层级显示缩进引导线；
- 语义缩略图展示标题、列表、代码、Git 改动、光标和当前视口；
- 可在缩略图上单击或拖动以快速滚动长文档。

## Finder Quick Look

- 在 Finder 中选中 Markdown 文件并按空格查看渲染预览；
- Quick Look 复用 FloralMD 的 HTML 渲染能力；
- 支持本地图片和主要 Markdown 语法；
- 默认阻止远程图片，并以可读源码保留数学公式。

## 外观与本地化

- 原生浅色与深色外观；
- 可配置正文、标题、代码和数学公式样式；
- 可调整字体、字号、行距、段落间距和物理内容宽度；
- 可开关文字抗锯齿、语义缩略图和打字机滚动；
- 应用界面支持英文和简体中文；
- AppKit 管理的保存提醒等系统界面跟随应用本地化声明。

## 本地优先与安全

- 文件默认在本地读取、编辑和保存；
- 阅读模式禁用 JavaScript；
- 对危险 HTML 标签、事件属性和脚本 URL 执行过滤；
- 外部网页链接交给系统默认浏览器处理；
- 自动更新功能在独立签名与发布配置完成前保持关闭。

## 当前限制

- FloralMD 仍处于测试阶段，界面和行为可能变化；
- 应用尚未使用 Apple Developer ID 签名，首次打开可能出现 Gatekeeper 提示；
- Git 面板当前用于查看状态，不提供暂存、提交、分支或图谱操作；
- Markdown 在不同解析器之间存在细节差异，FloralMD 主要遵循 CommonMark 与 GFM；
- Finder Quick Look 为安全起见不会加载远程图片。
