# FloralMD 功能说明

这里汇总 FloralMD 目前提供的主要功能。

## 文件与窗口

- 从 Finder、菜单或命令行直接打开任意 `.md`、`.markdown` 和支持的纯文本文件；
- 不要求文件属于 Vault、工作区或特定文件夹；
- 使用 macOS 原生窗口标签页，在同一窗口切换不同目录中的文档；
- `⌘N` 固定在当前窗口新建标签页，`⌘⇧N` 明确新建独立窗口，不依赖 macOS 的自动标签偏好；
- `⌘O` 默认在当前窗口以新标签打开文件，也可从文件菜单明确在新窗口中打开；
- “最近打开”实时列出可用文件，`⌃R` 可直接调出选择菜单，最近项同样在当前标签组打开；
- 冷启动、`⌘N` 和快速记录创建的空白未命名文档会显示低打扰的输入提示、最近文件与
  “打开文件”入口；开始输入时提示淡出，删除回空白时恢复，保存后的文档不再显示；
- 支持自动保存、撤销、重做与未保存修改提醒；
- 支持从 Finder 双击 Markdown 文件打开 FloralMD。

## 悬浮便签与快速记录

- 任意文档窗口都可独立置顶，持续显示在其他应用上方；
- 置顶窗口使用固定的半透明背景，同时保持文字、光标、控件、图片和公式清晰；
- 可为快速记录配置系统级快捷键，从任何应用唤起紧凑的置顶草稿；
- 快速记录复用普通 Markdown 文档流程，可配合未命名草稿自动落盘，不引入专有便签格式。

## 编辑与阅读

- 行内实时渲染 Markdown，光标进入当前块时显示可编辑的原始标记；
- 提供编辑模式、源码模式和阅读模式；
- 源码模式使用等宽字体展示完整 Markdown；
- 阅读模式使用独立 HTML 渲染，并随文档修改和主题变化刷新；
- 支持查找、替换、拼写检查、文本缩放和常用 macOS 编辑快捷键；
- 支持 PDF 导出与系统打印。

## Markdown 语法

- 标题、粗体、斜体、粗斜体、删除线和高亮；
- 有序列表、无序列表、任务列表与嵌套列表；
- 引用、Callout、分隔线和脚注；
- 行内代码、围栏代码块与语法高亮；
- GFM 表格，包括列对齐、独立单元格换行和长字符串折行；
- 普通链接、自动链接、引用链接、图片和 Wiki Link；
- 行内数学公式与块级数学公式；
- HTML 标签与经过限制的原始 HTML 阅读渲染；
- 反斜杠转义、注释和常用 GFM 扩展。

## 文档导航

- 左侧大纲根据当前文档标题生成，并跟随阅读位置；
- 大纲可折叠为窄边缘入口；
- 右侧文件树显示当前 Git 工作区或当前目录中的 Markdown 文件；
- 右侧侧栏可拖动调整宽度，并在当前文档标签会话内保留展开宽度与浏览模式；
- 文件夹显示递归 Markdown 数量，并自动展开到当前文件；
- 支持在文件树中切换文档；
- 文件树右键菜单支持打开、重命名、在 Finder 中显示、复制路径，以及将 Markdown
  文件可恢复地移到废纸篓；
- Git 状态颜色会从改动文件传播到父文件夹。

## Git 与长文档辅助

- 自动向上查找距离当前文件最近的 Git 仓库；
- 展示已修改、新增、删除和未跟踪文件；
- 可在 Git 面板、文件菜单或命令面板中确认仓库、分支、相对路径和状态，保存后只创建
  当前 Markdown 文件的本地提交；其他文件的暂存与未暂存状态保持不变，且不会推送；
- 只读展示当前仓库最近 80 个提交的本地分支与合并拓扑，并标出 HEAD、分支名、
  短哈希、提交信息、作者和时间；跨分支连接使用平滑曲线，HEAD 在提交行中明确标出；
- 选择提交可查看完整哈希、父提交和本地分支，并复制哈希；
- 编辑区左侧显示相对当前提交的改动标记；
- 列表层级显示缩进引导线；
- 语义缩略图展示标题、列表、代码、Git 改动、光标和当前视口；
- 可在缩略图上单击或拖动以快速滚动长文档。

## Finder Quick Look

- 在 Finder 中选中 Markdown 文件并按空格查看渲染预览；
- Quick Look 复用 FloralMD 的 HTML 渲染能力；
- 支持本地图片和主要 Markdown 语法；
- 默认阻止远程图片，并以可读源码保留数学公式。

## 外观与本地化

- 原生浅色与深色外观；
- 可配置正文、标题、代码和数学公式样式；
- 可调整字体、字号、行距、段落间距和物理内容宽度；
- 可开关文字抗锯齿、语义缩略图和打字机滚动；
- 应用界面支持英文和简体中文；
- AppKit 管理的保存提醒等系统界面跟随应用本地化声明。

## 本地优先与安全

- 文件默认在本地读取、编辑和保存；
- 阅读模式禁用 JavaScript；
- 对危险 HTML 标签、事件属性和脚本 URL 执行过滤；
- 外部网页链接交给系统默认浏览器处理；
- 自动更新通过 Sparkle 的签名更新源检查正式版本，也可以从应用菜单手动触发。

## 当前限制

- 应用尚未使用 Apple Developer ID 签名，首次打开可能出现 Gatekeeper 提示；
- Git 面板只增加了“提交当前文件”这一条窄操作，并只读展示工作区状态与有限的本地
  提交历史；不提供通用暂存、push、checkout、rebase、merge、冲突处理、diff 或完整
  历史浏览；
- Markdown 在不同解析器之间存在细节差异，FloralMD 主要遵循 CommonMark 与 GFM；
- Finder Quick Look 为安全起见不会加载远程图片。
